01

Qu'est-ce que le HTML ?

HTML signifie HyperText Markup Language, c'est-à-dire « langage de balisage hypertexte ». C'est le langage de base de toute page web : il sert à structurer le contenu qui s'affiche dans un navigateur (Chrome, Firefox, Edge…).

Premier point essentiel à retenir : le HTML n'est pas un langage de programmation. On ne lui demande pas de « calculer » ou de prendre des décisions. C'est un langage de balisage : on entoure le texte avec des marques (les balises) pour indiquer au navigateur la nature de chaque élément. Ceci est un titre, ceci est un paragraphe, ceci est une image, ceci est un lien.

Une comparaison simple

Pour construire une page web moderne, on combine trois technologies. L'analogie de la maison aide à les distinguer :

  • HTML : le gros œuvre, les murs et les pièces. C'est la structure du contenu.
  • CSS : la peinture, le carrelage, la décoration. C'est la présentation (couleurs, polices, mise en page).
  • JavaScript : l'électricité et les automatismes. C'est le comportement (réagir à un clic, animer, calculer).

Dans ce cours, nous nous concentrons uniquement sur le HTML, le socle indispensable sur lequel reposent les deux autres.

02

Un peu d'histoire

Le HTML est né à la fin des années 1980 au CERN (le Centre européen de recherche nucléaire, à Genève). Un chercheur britannique, Tim Berners-Lee, cherchait un moyen de partager facilement des documents entre scientifiques travaillant sur des machines différentes.

En 1991, il publie les premières spécifications du HTML. Sa grande idée est le lien hypertexte : la possibilité de cliquer sur un mot pour sauter vers un autre document. C'est cette idée, simple mais révolutionnaire, qui a donné naissance au World Wide Web tel que nous le connaissons.

La toute première version ne comptait qu'une vingtaine de balises. Depuis, le langage a beaucoup évolué, encadré par deux organismes :

  • le W3C (World Wide Web Consortium), créé en 1994, qui définit les standards officiels du Web ;
  • le WHATWG, un groupe d'éditeurs de navigateurs qui maintient aujourd'hui le HTML sous forme de « standard vivant » (mis à jour en continu).

Les grandes dates

AnnéeÉtape clé
1991Tim Berners-Lee publie le premier HTML au CERN. Le Web naît.
1993Le navigateur Mosaic popularise le Web et introduit l'affichage des images.
1995HTML 2.0 : premier standard officiel, avec les formulaires.

2000XHTML 1.0 : une syntaxe plus stricte, inspirée du XML.

03

Anatomie d'une balise

Une balise (en anglais tag) est un mot-clé encadré par des chevrons < >. C'est l'unité de base du HTML. La plupart des balises fonctionnent par paire : une balise ouvrante et une balise fermante qui encadrent un contenu.

Les trois parties d'un élément

Observons l'exemple suivant, qui affiche un message à l'écran :

<p class="message">Bonjour Dakar</p>
Balise ouvrante<p> marque le début. Ici p = paragraphe.
Attributclass="message" : un réglage, sous la forme nom="valeur".
Contenu« Bonjour Dakar » : le texte réellement affiché.
Balise fermante</p> marque la fin. Notez la barre oblique /.

L'ensemble « balise ouvrante + contenu + balise fermante » forme ce qu'on appelle un élément HTML.

Les attributs

Dans l'exemple, class="message" est un attribut. Un attribut ajoute une information ou un réglage à la balise. Il se place toujours dans la balise ouvrante et suit la forme nom="valeur". Les attributs servent par exemple à donner une adresse à un lien, une source à une image, ou un identifiant à un élément.

04

L'évolution du HTML, niveau par niveau

Le HTML n'est pas né complet : il s'est enrichi par étapes, chaque version corrigeant les limites de la précédente. Nous allons parcourir cinq « niveaux » qui correspondent aux grandes versions historiques. Pour bien percevoir l'évolution, nous suivrons un fil conducteur : la présentation d'une petite boutique d'électronique de Dakar, Teranga Électronique.

01
HTML 1.0 · 1991–1993

Les débuts

Au tout début, le HTML servait à publier de simples documents : un titre, des paragraphes et surtout des liens. La mise en forme était quasi inexistante. Voici à quoi ressemblait une page minimale :

Ce que l'on peut faireStructurer un document avec des titres (<h1>), des paragraphes (<p>) et des liens (<a>).
Les limitesAucun contrôle visuel, pas de tableaux, pas de formulaires. La page est purement textuelle.
Pourquoi évoluer ?Le Web grandit vite. Les utilisateurs veulent envoyer des informations (s'inscrire, rechercher) et pas seulement lire. Il faut rendre les pages interactives.
02
HTML 2.0 · 1995

La standardisation

HTML 2.0 est le premier vrai standard officiel. Son apport majeur est le formulaire, qui permet à l'internaute de saisir des données et de les envoyer à un serveur. La boutique peut désormais proposer un formulaire d'inscription :

Ce que l'on gagneL'interactivité. Avec <form> et <input>, le visiteur n'est plus passif, il peut communiquer avec le site.
Les limitesLe langage reste pauvre visuellement. On ne peut ni mettre en page proprement, ni présenter des données en colonnes.
Pourquoi évoluer ?Les sites veulent un rendu plus soigné : des couleurs, des tableaux, une mise en page organisée.
03
HTML 3.2 · 1997

La mise en forme

HTML 3.2 introduit les tableaux et de nombreuses balises de présentation comme <font>, <center> ou l'attribut bgcolor. On peut enfin colorer et organiser le contenu :

Ce que l'on gagneUn vrai contrôle visuel et la possibilité d'afficher des données en tableau (idéal pour une liste de prix).
Le problèmeLa mise en forme est mélangée au contenu. Couleur, police et alignement sont écrits dans chaque balise. Sur un grand site, changer une couleur oblige à modifier des centaines de balises <font>. Le code devient lourd et difficile à maintenir.
Pourquoi évoluer ?Il faut absolument séparer le contenu de sa présentation.
04
HTML 4.01 · 1999

La séparation contenu / présentation

C'est une étape décisive. HTML 4.01 confie toute la mise en forme à une technologie séparée, le CSS (Cascading Style Sheets). Le HTML décrit ce qu'est le contenu ; le CSS décrit à quoi il ressemble. Les balises de présentation comme <font> sont abandonnées.

Les couleurs et les polices ne sont plus dans la page : elles sont décrites une seule fois dans le fichier styles.css, relié par la balise <link>. On utilise des attributs class pour cibler les éléments à styler. Cette version apporte aussi une vraie structure de document avec <html>, <head> et <body>.

Ce que l'on gagneUn code propre et maintenable. Changer toute l'apparence du site se fait en modifiant un seul fichier CSS.
Les limitesLe HTML 4 reste « générique ». Tout est construit avec des <div> sans signification précise. De plus, le Web devient multimédia (vidéo, audio) et mobile, ce que cette version gère mal.
Pourquoi évoluer ?Il faut donner du sens à la structure et intégrer nativement la vidéo, l'audio et les usages mobiles.
05
HTML5 · 2014

Le Web moderne

HTML5 est la version utilisée aujourd'hui. Elle apporte deux grandes nouveautés. D'abord les balises sémantiques : au lieu de <div> partout, on dispose de balises qui décrivent leur rôle, comme <header> (en-tête), <nav> (menu), <main> (contenu principal), <section>, <article> et <footer> (pied de page). Ensuite, le multimédia natif avec <video> et <audio>, qui rendent les anciens modules externes inutiles.

Ce que l'on gagneUn code clair et compréhensible, un meilleur référencement (Google comprend mieux la page), une bonne accessibilité, et le support natif de la vidéo, de l'audio et des écrans mobiles.
Pourquoi c'est le standardHTML5 répond à tous les usages actuels du Web et continue d'évoluer en permanence. C'est la version que vous apprendrez et utiliserez.

05

La structure d'une page HTML5

Toute page moderne suit le même squelette. Voici le modèle de départ, ligne par ligne :

  • <!DOCTYPE html> : indique au navigateur que la page est écrite en HTML5. Toujours en première ligne.
  • <html lang="fr"> : la racine de la page. L'attribut lang="fr" précise que le contenu est en français.
  • <head> : l'en-tête « invisible ». Il contient les informations techniques sur la page, pas le contenu affiché.
  • <meta charset="UTF-8"> : définit l'encodage des caractères. Indispensable pour afficher correctement les accents (é, è, à…).
  • <meta name="viewport" …> : assure un bon affichage sur les téléphones et tablettes.
  • <title> : le texte affiché dans l'onglet du navigateur.
  • <body> : le corps de la page. Tout ce qui est visible par le visiteur se place ici.
  • <!-- … --> : un commentaire. Le navigateur l'ignore ; il sert de note pour le développeur.

CSS inline et CSS interne

Le CSS (vu en section 1) peut s'écrire à plusieurs endroits différents, et on peut viser un élément précis avec deux attributs : class ou id. Regardons le CSS inline et le CSS interne à partir de deux exemples bien réels, tirés de cette page : le petit bloc « HTML » du logo (avec class), et l'en-tête « Terme » du glossaire (avec id).

Le CSS inline

Le CSS inline consiste à écrire le style directement dans la balise, à l'intérieur d'un attribut style="...". C'est la façon la plus directe d'appliquer un style à un élément précis.

Ce que l'on gagneRapide à écrire pour styler un seul élément, sans toucher à autre chose.
La limiteLe style est mélangé au contenu — exactement le problème déjà rencontré avec la balise <font> en HTML 3.2 (section 4). Si dix éléments doivent partager ce style, il faut le réécrire dix fois.

Passer au CSS interne : la méthode

Le CSS interne regroupe tous les styles dans une seule balise <style>, placée dans le <head> (c'est exactement la grosse balise <style> qui se trouve en haut du code de cette page). Pour transformer un style inline en CSS interne, on suit toujours les mêmes étapes :

  1. On repère le texte écrit dans l'attribut style="..." de la balise.
  2. On coupe ce texte (on le retire de l'attribut).
  3. On remplace l'attribut style="..." par un attribut class="nom-de-classe" — un nom qui décrit le rôle de l'élément.
  4. Dans la balise <style>, on écrit .nom-de-classe suivi d'une paire d'accolades { }.
  5. On colle, à l'intérieur de ces accolades, le style coupé à l'étape 2.

Le second attribut possible : id

Pour cibler un élément, on peut utiliser class… ou id. La règle à retenir : un id doit être unique — une seule fois dans toute la page — alors qu'une class peut être posée sur autant d'éléments qu'on veut. Dans la balise <style>, on écrit le sélecteur avec un point pour une classe (.nom-de-classe) et avec un croisillon (#) pour un id (#nom-d-id) — la méthode de conversion reste identique, seul ce symbole change.
